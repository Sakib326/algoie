# Day 2 Additional — Production-Ready Authentication, Public Storefront & Application Foundation

## Overview

Replace all temporary `actor: :system` implementations with real authentication, build a professional dashboard shell, public landing page, storefront foundation, and custom error pages. This makes the app usable as a real SaaS product.

---

## Architecture Summary

### How Authentication Integrates

The project already has AshAuthentication configured on the User resource with a password strategy. AshAuthenticationPhoenix 2.17.1 provides ready-made LiveView components for sign-in, sign-up, password reset, and sign-out. We use these directly.

**Session flow:**
1. User visits `/sign-in` → AshAuthenticationPhoenix.SignInLive renders email+password form
2. Form submits to `/auth/user/password/sign_in` → AuthController.success/4 stores user in session
3. `load_from_session` plug (in browser pipeline) retrieves user from session on every request
4. Dashboard `live_session` uses `AshAuthentication.Phoenix.LiveSession` on_mount hook to load `current_user` into socket assigns
5. All dashboard LiveViews use `actor: @current_user` instead of `actor: :system`

**Three route groups:**
- **Public** (`/`): Landing page, sign-in, sign-up, password reset — uses `Layouts.app`
- **Dashboard** (`/dashboard/*`): Authenticated CRUD — uses `Layouts.dashboard` with sidebar
- **Storefront** (`/` on subdomain): Public product browsing — uses `Layouts.storefront`

### Step 2: Create AuthController

**New file: `lib/algoie_web/controllers/auth_controller.ex`**

```elixir
defmodule AlgoieWeb.AuthController do
  use AlgoieWeb, :controller
  use AshAuthentication.Phoenix.Controller

  alias Algoie.Accounts.User

  def success(conn, _activity, user, _token) do
    conn
    |> store_in_session(user)
    |> redirect(to: "/dashboard")
  end

  def failure(conn, _activity, _reason) do
    conn
    |> put_flash(:error, "Invalid email or password")
    |> redirect(to: "/sign-in")
  end

  def sign_out(conn, _params) do
    conn
    |> clear_session(:algoie)
    |> put_flash(:info, "Signed out successfully")
    |> redirect(to: "/")
  end
end
```

### Step 3: Router Overhaul

**File to modify: `lib/algoie_web/router.ex`**

Complete rewrite of the router structure:

```elixir
defmodule AlgoieWeb.Router do
  use AlgoieWeb, :router
  use AshAuthentication.Phoenix.Router

  pipeline :browser do
    plug :accepts, ["html"]
    plug :fetch_session
    plug :fetch_live_flash
    plug :put_root_layout, html: {AlgoieWeb.Layouts, :root}
    plug :protect_from_forgery
    plug :put_secure_browser_headers
    plug :load_from_session
  end

  pipeline :api do
    plug :accepts, ["json"]
  end

  pipeline :store do
    plug AlgoieWeb.Plugs.StoreSlugPlug
  end

  # ── Public routes (no auth required) ──
  scope "/", AlgoieWeb do
    pipe_through :browser

    # Landing page
    live "/", HomeLive, :index

    # Authentication
    sign_in_route(
      path: "/sign-in",
      auth_routes_prefix: "/auth",
      register_path: "/register",
      overrides: [AshAuthentication.Phoenix.Overrides.DaisyUI]
    )
    sign_out_route(AuthController, "/sign-out")
    auth_routes(AuthController, Algoie.Accounts.User, path: "/auth")
    reset_route(auth_routes_prefix: "/auth", overrides: [AshAuthentication.Phoenix.Overrides.DaisyUI])
  end

  # ── Dashboard routes (auth required) ──
  scope "/", AlgoieWeb do
    pipe_through [:browser]

    ash_authentication_live_session :dashboard,
      otp_app: :algoie,
      on_mount_prepend: [{AlgoieWeb.Live.OnDashboardMount, :default}] do

      live "/dashboard", DashboardLive, :index
      live "/dashboard/products", ProductLive.Index, :index
      live "/dashboard/products/new", ProductLive.Index, :new
      live "/dashboard/products/:id/edit", ProductLive.Index, :edit
      live "/dashboard/categories", CategoryLive.Index, :index
      live "/dashboard/categories/new", CategoryLive.Index, :new
      live "/dashboard/categories/:id/edit", CategoryLive.Index, :edit
      live "/dashboard/brands", BrandLive.Index, :index
      live "/dashboard/brands/new", BrandLive.Index, :new
      live "/dashboard/brands/:id/edit", BrandLive.Index, :edit
      live "/dashboard/orders", OrderLive.Index, :index
      live "/dashboard/orders/:id", OrderLive.Show, :show
    end
  end

  # ── Storefront routes (subdomain-based, no auth required) ──
  scope "/", AlgoieWeb do
    pipe_through [:browser, :store]

    live "/", StorefrontHomeLive, :index
    live "/products", StorefrontProductLive.Index, :index
    live "/products/:id", StorefrontProductLive.Show, :show
  end

  # ── API routes ──
  scope "/api", AlgoieWeb do
    pipe_through [:api]
  end
end
```

### Step 4: OnDashboardMount Hook

**New file: `lib/algoie_web/live/on_dashboard_mount.ex`**

Loads `current_user` from session via AshAuthenticationPhoenix's LiveSession hook, plus loads the user's store context. Redirects to sign-in if not authenticated.

```elixir
defmodule AlgoieWeb.Live.OnDashboardMount do
  @moduledoc false

  import Phoenix.LiveView
  import Phoenix.LiveView.Helpers, only: [redirect: 2]

  def on_mount(:default, _params, _session, socket) do
    if socket.assigns[:current_user] do
      # Load the user's store context from their StoreStaff membership
      user = socket.assigns.current_user
      {:cont, assign(socket, :current_scope, %{user: user})}
    else
      {:halt, redirect(socket, to: "/sign-in")}
    end
  end
end
```
